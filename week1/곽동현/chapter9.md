## 운영체제란?
- 컴퓨터 부품을 동작 시키기 위한 프로그램이다.
    - 프로그램 : 메모리에 올라가서 사용되는 소프트웨어
    - 하지만 운영체제는 너무 중요하기 떄문에, 일반 응용프로그램 처럼 단순 메모리에 올라가지 않고,
      `커널 영역` 에 올라간다.
- 단순 동작시키는 것 뿐만 아니라 다양한 일을 담당한다.
- 현존하는 프로그램 규모 중 가장 큰 프로그램 중 하나이다. (코드만 2700만줄 이상)

### 운영체제의 핵심 서비스 1 : 프로세스 관리
- 동시에 실행중인 프로그램듣을 관리한다.
- 동시다발적으로 생성/실행/삭제 되는 프로세스들을 일목요연하게 관리한다.(시분할 시스템)

### 운영체제의 핵심 서비스 2 : 자원 접근 및 할당
- CPU 자원 할당
    - CPU 스케줄링 담당 : 어떤 프로세스를 먼저, 얼마나 오랫동안 실행시킬지 결정
- 메모리 자원 할당
    - 페이징 / 스와핑 담당 : 비어있는 메모리 공간을 어떻게 찾고, 어디에 할당할 지 프로그램 적재 + 메모리에 말고 다른곳에서 실행되는 방법
- 입출력 장치 자원 할당
    - 인터럽트 서비스 루틴 제공 : 프로그램들이 입출력 장치에 접근하고자 할 때, 접근해서 실행되고자 할 때 할당한다.

### 운영체제의 핵심 서비스 3 : 파일 시스템 관리
- 파일 / 폴더 관리 : 파일 / 폴더도 다 운영체제가 제공해주는 기능이다.
- 보조 기억 장치에 있는 정보 덩어리(파일/폴더)를 하나의 단위로 묶어서 관리된다.
- 운영체제가 이 파일/폴더를 저장시키고, 보안적으로 관리한다.

### 지금까지의 내용을 종합해보면..

- 결국 운영체제는 여러 프로세스(응용프로그램) 과 하드웨어 장치를 이어주는 , `연결다리` 와 같은 역할을 한다.

## 커널이란?
- 운영체제의 핵심 서비스를 담당하는 부분이다.
- 모든 운영체제는 반드시 하나의 커널을 가지고 있어야한다.
- 학습중에는 운영체제 == 커널 이라고 봐도 무방하다.

## 운영체제 이지만, 커널은 아닌것은?
- GUI(Graphic User Interface) : 사용자 인터페이스
- CLI(Command Line Interface) : 커맨드 라인 인터페이스

## 이중 모드란?
- CPU 가 명령어를 실행하는 모든 모드를 크게 `사용자 모드` / `커널 모드` 로 구분하는 방식이다.
- 운영체제의 서비스를 받을 수 있느냐 / 없느냐로 구분된다.
    - 운영체제 서비스 : 자원 접근

### 사용자 모드
- 운영체제의 서비스를 제공 받을 수 없는 실행모드
- 따라서, 사용자 모드의 응용프로그램은 자원에 접근할 수 없다.

### 커널 모드
- 운영체제의 서비스를 제공 받을 수 있는 실행모드
- 따라서, 커널 모드의 응용프로그램은 자원에 접근할 수 있다.

## 언제 사용자 모드 -> 커널 모드로 바뀌는가?
- 시스템 호출(System Call)을 하게 되면, 사용자모드 -> 커널 모드로 변경된다.
- ex) 입출력 장치를 이용하면 시스템 콜이 발생되고, 자원 할당이 가능하다.

## 시스템 호출(System Call)
- 시스템 호출은 일종의 인터럽트(Interrupt) 이다.
- 인터럽트는 입출력 장치에 의해 발생하기도 하지만, 인터럽트를 발생시키는 특정 명령어에 의해 발생하기도 한다.
    - 이를 `소프트웨어 인터럽트` 라고 한다.
