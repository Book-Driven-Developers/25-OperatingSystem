## CPU 스케줄링이란?
- 운영체제가 여러 프로세스들에게 공정하고 합리적으로 CPU 자원을 배분하는 것
- 컴퓨터의 성능과도 직결되는 중요한 요소이다.

## 프로세스 우선순위
- 프로세스 우선순위는 각 PCB에 저장된다.

### 프로세스 우선순위 예시
- 입출력 작업이 많은 프로세스가 CPU 작업이 많은 프로세스보다 우선순위가 높다.
    - 입출력 작업이 많다 => 대기시간이 많아 질 것이다 => 얼른 처리해버리고 CPU 작업이 많은 프로세스에 몰빵하자.

## 스케줄링 큐
- CPU 가 PCB 의 우선순위를 찾기 위해 매번 완전탐색을 할 수 없기 때문에, PCB를 스케줄링 큐로 관리한다.
- CPU 스케줄링 큐 / HDD 스케줄링 큐 / 입출력장치(프린터) 스케줄링 큐 등 각각의 스케줄링 큐가 존재한다.
- 스케줄링 큐는 선입선출(FIFO)은 아니다.

## 스케줄링 큐 대표 2가지
### 준비 큐
- CPU를 이용하고자 하는 프로세스들이 서는 줄
- 실행 시간 완료 이벤트(타이머 인터럽트)가 발생하여, 준비 상태가 접어들어 서는 줄

### 대기 큐
- 입출력 장치를 이용하고자 하는 프로세스들이 서는 줄
- 입출력 완료 이벤트(인터럽트)가 발생하여, 대기 상태에 접어들어 서는 줄

## 선점형 / 비선점형 스케줄링

### 선점형 스케줄링
- 현재 CPU를 사용중인 프로세스로부터 CPU 자원을 **빼앗아** 다른 프로세스에 할당
- 장점 : 어느 한 프로세스의 자원 독점을 막고, 프로세스들에 골고루 자원을 배분할 수 있다.
- 단점 : 문맥 교환 과정에서 오버헤드가 발생할 수 있다.

### 비선점형 스케줄링
- 현재 CPU를 사용중인 프로세스의 작업이 끝날 때까지 프로세스 **기다리기**
- 장점 : 선점형 스케줄링에 비해 문맥 교환에서 발생하는 오버헤드가 적다.
- 단점 : 모든 프로세스가 골고루 자원을 이용하기 어렵다.

<summary> CPU 스케줄링 알고리즘 </summary>

## FCFS : 선입 선처리 스케줄링
- 단순히 준비 큐에 삽입된 순서대로 처리하는 비선점 스케줄링
- 먼저 CPU를 요청한 프로세스부터 CPU 할당
- 단점 : 프로세스들이 기다리는 시간이 매우 길어질 수 있다는 부작용이 있다. (=호위 효과)
    - 앞 순서의 프로세스가 정말 긴 실행시간 보유 할 때

## SJF : 최단 작업 우선 스케줄링
- CPU 사용이 긴 프로세스는 나중에 실행, CPU 사용 시간이 짧은 프로세스는 먼저 실행
- CPU 사용 시간이 가장 짧은 프로세스부터 처리하는 스케줄링 방식

## RR : 라운드 로빈 스케줄링
- 선입 선처리 스케줄링 + 타임 슬라이스
- 타임 슬라이스 : 각 프로세스가 CPU 를 사용할 수 있는 정해진 시간
- 정해진 타임 슬라이스만큼의 시간 동안 돌아가며, CPU 를 이용하는 **선점형 스케줄링**
    - 선입 선처리 대로 하되 제한된 타임 슬라이스 동안 못끝내면 queue 뒤로 이동 (문맥 교환)

## SRT : 최소 잔여 시간 우선 스케줄링
- 최단 작업 우선 스케줄링 + 라운드 로빈 스케줄링
- 정해진 시간만큼 CPU를 이용하되, 다음으로 CPU를 사용할 프로세스로는 남은 작업 시간이 가장 적은 프로세스 선택

## 우선순위 스케줄링
- 프로세스들에 우선순위를 부여하고, 우선순위가 높은 프로세스부터 실행
- 우선순위가 같은 프로세스들은 선입 선처리로 스케줄링
- 최단 작업 우선 스케줄링, 최소 잔여 시간 스케줄링 <= 우선순위 스케줄링
- 단점 : 우선순위 스케줄링의 근본적인 문제점. **기아(starvation)** 현상 발생
    - 우선 순위 높은 프로세스만 지속적으로 실행
    - 우선 순위가 낮은 프로세스는 실행이 영원히 실행이 못 될수도 있음.
- 해결방안 : **에이징(aging)**
    - 오랫동안 대기한 프로세스의 우선순위를 점차 높이는 방식
    - 대기 중인 프로세스의 우선순위를 마치 나이를 먹듯 점차 증가시키는 방법
        - 우선순위가 낮아도 언젠가는 우선순위가 높아질 수 있다.

## MultiLevel Queue : 다단계 큐 스케줄링
- 우선순위 스케줄링의 발전된 형태
- 각 큐마다 서로 다른 스케줄링 알고리즘을 적용할 수도 있다.
- 우선 순위별로 준비 큐를 여러 개 사용하는 스케줄링 방식이다.
    - 우선순위가 가장 높은 큐에 있는 프로세스를 먼저 처리
    - 우선순위가 가장 높은 큐가 비어 있으면, 그 다음 우선순위 큐에 있는 프로세스가 처리된다.
- 단점 : 큐 간의 이동이 불가하다.
    - 기아(starvation) 현상이 발생할 수 있다.

## MultiLevel Feedback Queue : 다단계 피드백 큐 스케줄링
- 다단계 큐 스케줄링의 발전된 형태
- 각 큐마다 이동이 가능하다.
- 타임 슬라이스 동안 실행을 끝내지 못한다면, 낮은 우선순위로 점차 이동한다.
    - 입출력 집중 프로세스의 우선순위는 높아지고, 먼저 끝낼 수 있다.
    - CPU 집중 프로세스의 우선순위는 자연스럽게 낮아질 수 있다.
- 에이징 기법또한 적용할 수 있어서, 너무 낮아진 우선순위를 다시 높힐 수 있다.
