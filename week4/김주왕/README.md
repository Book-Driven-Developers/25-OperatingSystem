## 1. 교착 상태란?

### 1.1 교착 상태 (Deadlock)

**교착 상태(deadlock)** 란  
일어나지 않을 사건을 기다리며 프로세스의 실행이 영원히 멈춰 버리는 현상을 의미한다.

- 여러 프로세스가 서로의 자원을 기다림
- 어느 프로세스도 먼저 진행할 수 없음
- CPU는 놀고 있지만 시스템은 멈춘 상태

---

### 1.2 식사하는 철학자 문제 (Dining Philosophers Problem)

교착 상태를 설명하는 대표적인 고전 문제이다.

- 철학자 N명과 포크 N개
- 철학자는 **왼쪽 포크와 오른쪽 포크를 모두 들어야** 식사 가능
- 모든 철학자가 동시에 왼쪽 포크를 집으면  
  → 서로 오른쪽 포크를 기다리며 교착 상태 발생

---

### 1.3 자원 할당 그래프 (Resource Allocation Graph)

교착 상태를 시각적으로 표현하는 방법

- 프로세스: 원 (○)
- 자원 종류: 사각형 (□)
- 자원 개수: 사각형 내부의 점
- 자원 할당: 자원 → 프로세스
- 자원 요청: 프로세스 → 자원

그래프에 **사이클(cycle)** 이 존재하면 교착 상태 가능성이 존재한다.

---

### 1.4 교착 상태 발생의 4가지 조건

아래 조건이 **모두 만족될 때** 교착 상태가 발생할 수 있다.

1. **상호 배제 (Mutual Exclusion)**  
   한 프로세스가 사용하는 자원을 다른 프로세스가 동시에 사용할 수 없음

2. **점유와 대기 (Hold and Wait)**  
   자원을 보유한 상태에서 다른 자원을 기다림

3. **비선점 (No Preemption)**  
   자원을 강제로 회수할 수 없음

4. **원형 대기 (Circular Wait)**  
   프로세스들이 원형 구조로 자원을 대기

---

### 핵심 포인트 (4 Keywords)

- 교착 상태는 무한 대기 현상이다
- 식사하는 철학자 문제는 대표적인 예시이다
- 자원 할당 그래프로 표현할 수 있다
- 발생 조건은 상호 배제, 점유와 대기, 비선점, 원형 대기이다

---

## 2. 교착 상태 해결 방법

운영체제는 교착 상태를 다음 네 가지 전략 중 하나로 다룬다.

---

## 2.1 교착 상태 예방 (Deadlock Prevention)

교착 상태 발생 조건 중 **하나라도 성립하지 않게** 만드는 방식이다.

### 주요 방법

#### 1) 상호 배제 제거
- 모든 자원을 공유 가능하게 만듦
- 현실적으로 불가능

#### 2) 점유와 대기 제거
- 프로세스 시작 시 필요한 모든 자원을 한 번에 할당
- 단점
  - 자원 활용률 감소
  - 기아 현상 발생 가능

#### 3) 비선점 제거
- 자원을 강제로 회수 가능
- CPU 같은 자원에는 효과적
- 파일, 프린터 등에는 부적절

#### 4) 원형 대기 제거
- 모든 자원에 번호 부여
- 오름차순으로만 자원 할당
- 실무에서 가장 현실적인 예방 기법

> 예방 방식은 교착 상태를 원천 차단하지만 성능 저하라는 비용이 존재한다.

---

## 2.2 교착 상태 회피 (Deadlock Avoidance)

교착 상태가 **발생하지 않을 정도까지만** 자원을 할당하는 방식이다.

### 핵심 개념

- **안전 상태 (Safe State)**  
  모든 프로세스가 정상 종료 가능한 상태

- **불안전 상태 (Unsafe State)**  
  교착 상태가 발생할 수도 있는 상태

- **안전 순서열 (Safe Sequence)**  
  교착 상태 없이 프로세스를 실행할 수 있는 순서

운영체제는 항상 **안전 상태 → 안전 상태**로만 이동하도록 자원을 할당한다.

대표 알고리즘: **은행원 알고리즘 (Banker’s Algorithm)**

---

## 2.3 교착 상태 검출 후 회복 (Detection & Recovery)

교착 상태 발생을 허용하고, 사후에 해결하는 방식이다.

### 검출
- 자원 할당 그래프 / 대기 그래프(WFG) 분석
- 사이클 존재 여부 검사

### 회복 방법
- 자원 선점
- 프로세스 강제 종료 (전체 또는 일부)

---

## 2.4 교착 상태 무시

- **타조 알고리즘 (Ostrich Algorithm)**
- 교착 상태는 드물다고 가정하고 무시
- UNIX, Windows 일부 전략

---

### 핵심 포인트 (3 Keywords)

- 예방: 조건 자체를 제거
- 회피: 안전 상태 유지
- 검출/회복: 발생 후 처리

---

## 3.1 `dining_deadlock.py`

### Purpose
식사하는 철학자 문제를 통해 **교착 상태를 의도적으로 발생**시키는 예제이다.

### Key Idea
- 모든 철학자가 **왼쪽 포크 → 오른쪽 포크** 순서로 자원 획득
- 원형 대기 조건을 의도적으로 만족시킴

### How It Works
1. 각 철학자는 왼쪽 포크를 먼저 획득
2. 잠시 대기하여 모든 철학자가 왼쪽 포크를 들도록 유도
3. 오른쪽 포크를 요청하면서 무한 대기 발생 가능

### Key Point
- 교착 상태 발생 조건 4가지가 모두 충족됨
- 출력이 멈추면 교착 상태 발생

---

## 3.2 `dining_prevent_ordered.py`

### Purpose
**원형 대기 조건 제거**를 통해 교착 상태를 예방하는 예제이다.

### Key Idea
- 자원(포크)에 전역 순서 부여
- 항상 작은 번호 → 큰 번호 순서로 획득

### How It Works
1. 두 포크 중 번호가 작은 것을 먼저 획득
2. 동일한 규칙을 모든 스레드에 적용
3. 원형 대기 구조 자체가 불가능

### Key Point
- 교착 상태 예방 기법 중 가장 현실적인 방식

---

## 3.3 `bankers_avoidance.py`

### Purpose
은행원 알고리즘을 사용하여 **교착 상태 회피**를 구현한다.

### Key Idea
- 안전 순서열 존재 여부 판단
- 안전 상태에서만 자원 할당 허용

### How It Works
1. Available / Max / Allocation 기반으로 Need 계산
2. 프로세스가 종료 가능한지 시뮬레이션
3. 모든 프로세스가 종료 가능하면 안전 상태

### Key Point
- 안전 순서열이 존재하면 교착 상태 발생하지 않음

---

## 3.4 `detection_recovery_wfg.py`

### Purpose
**교착 상태 검출 후 회복** 전략을 구현한 예제이다.

### Key Idea
- Wait-For Graph(WFG)에서 사이클 탐지
- 사이클 존재 시 프로세스 제거로 회복

### How It Works
1. DFS 기반으로 WFG 사이클 탐색 (이 부분은 추가 리뷰 필요..)
2. 사이클 발견 시 희생 프로세스 선택
3. 해당 프로세스를 제거하여 교착 상태 해소

### Key Point
- 교착 상태 발생을 인정하고 사후 처리
